# Pixel Mapper — App Store Asset Checklist

App Store Connect is stricter than Play about screenshot sizes — they must match
exact device resolutions. You only need to provide the largest size for iPhone
and iPad; Apple downscales for smaller devices.

## App icon
| Asset | Spec | Notes |
|---|---|---|
| App Store icon | **1024 × 1024 px**, PNG, **no alpha**, no rounded corners | Apple rounds it for you. Reuse `../play/assets/icon-512.png` upscaled from the 1024 master `assets/icon/app_icon.png`. |

The in-app icons are generated by `flutter_launcher_icons` (already configured in
`pubspec.yaml`, `ios: true`). Run `dart run flutter_launcher_icons` on the build
machine to (re)generate the iOS icon set.

## Screenshots (required)
At least one set is required. Capture from the iOS Simulator on a Mac
(`flutter run` → Simulator, then **Cmd+S** saves a correctly-sized PNG), or via
`xcrun simctl io booted screenshot out.png`.

| Device class | Required size (portrait) | Simulator to use |
|---|---|---|
| iPhone 6.9" | **1320 × 2868** (or 1290 × 2796) | iPhone 16 Pro Max |
| iPhone 6.5" | 1242 × 2688 | iPhone 11 Pro Max (if supporting older) |
| iPad 13" | **2064 × 2752** | iPad Pro 13-inch (M4) |

- 2–10 screenshots per device class.
- Suggested set mirrors the Play set: **Review** (lead), Target, Scan, Export.
- The same caveat applies: Review/Export need seeded data and Scan needs a real
  display — see `../play/screenshots/README.md` for the throwaway-seed approach,
  which works identically on the iOS Simulator.

## App preview video (optional)
15–30 s, same resolutions as screenshots. Skip for v1.

## Where it goes
App Store Connect → your app → **(version)** → **App Previews and Screenshots**
(per device size) and **App Information** for icon/metadata.
